# Локальная многоагентная чат-система

Настольное Windows-приложение на русском языке для общения с несколькими автономными ИИ-агентами через локальные модели Ollama.

## Возможности

- Русскоязычный GUI на Tkinter с главным окном чата.
- Отправка сообщений всем агентам или выбранному агенту.
- Панель управления агентами: создание, просмотр через список, редактирование и удаление.
- Краткосрочная и долгосрочная память каждого агента.
- Автоматическое сохранение состояния, истории сообщений, событий, агентов и выбранной модели в JSON.
- Пауза и возобновление симуляции без потери состояния.
- Проверка наличия исполняемого файла Ollama и локального API `http://127.0.0.1:11434`.
- Обновление списка локальных моделей и выбор глобальной или индивидуальной модели агента.
- Устойчивый движок: ошибки модели и некорректные JSON-ответы логируются и не останавливают приложение.
- Возможность создания новых агентов действиями существующих агентов через структурированный ответ модели.

## Запуск

1. Установите Python 3.10+ для Windows.
2. Установите Ollama: <https://ollama.com/download/windows>.
3. Загрузите модель, например:

   ```powershell
   ollama pull llama3.1
   ```

4. Запустите приложение:

   ```powershell
   python main.py
   ```

После запуска нажмите **«Обновить список моделей»**, выберите модель и нажмите **«Применить модель»**. Затем нажмите **«Возобновить»**, чтобы агенты начали циклическую обработку сообщений.

## Хранение данных

Состояние сохраняется в `%APPDATA%\LocalMultiAgentChat\state.json`. Если файл повреждён, приложение создаёт резервную копию и запускается с начальным состоянием.

## Формат действий агента

Каждый агент получает промпт с требованием вернуть JSON:

```json
{
  "respond": true,
  "message": "сообщение",
  "recipient": "Все",
  "memory": "важный вывод",
  "state_update": {"goal": "", "system_prompt": ""},
  "create_agent": null
}
```

Если модель вернула некорректный JSON, ответ сохраняется как обычное сообщение, а ошибка не прерывает работу системы.
